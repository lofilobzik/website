SYSTEM_DOCS_V1.0

01. PROJECT_OVERVIEW

The Challenge: Control theory concepts like PID (Proportional-Integral-Derivative) loops are often taught as abstract equations, making it difficult to visualize how tuning parameters affects physical stability.

The Solution: I engineered a real-time, browser-based physics simulation. It models a control system fighting against stochastic disturbances (random impulses) and constant orbital forces, allowing users to intuitively "tune" the math behind the movement.

02. SOFTWARE_ARCHITECTURE

This is not a pre-made component: the physics engine and control logic were built from the ground up.

  • Frontend: Three.js (WebGL) for high-performance 3D rendering.
  • Control Logic: Custom JavaScript implementation of the PID algorithm, featuring Integral Anti-Windup and Derivative Smoothing to handle discrete-time signal noise.
  • Optimization: Implemented efficient render loops and low-latency state management to ensure 60FPS performance on mobile devices.

03. INFRASTRUCTURE_&_IOT

Unlike standard cloud hosting, this project is a self-hosted hybrid system running entirely on hardware in my home lab.

  • Host: Raspberry Pi 4B (Linux/Debian).
  • Network: Exposed securely to the public internet via a Tailscale Funnel (Reverse Proxy).
  • Data Pipeline: A custom Python service polls a DHT22 sensor via GPIO pins and writes asynchronous telemetry data to the frontend.

05. LEARNING_OUTCOMES

Building this required solving several cross-domain engineering problems:

  • Control Theory: Tuned $K_p$, $K_i$, and $K_d$ gains to eliminate steady-state error without inducing oscillation.
  • Systems Administration: Managed headless Linux deployment, file permissions, and automated build scripts.
  • Full Stack Integration: Bridged the gap between hardware sensors (Python) and the user interface (JavaScript).
